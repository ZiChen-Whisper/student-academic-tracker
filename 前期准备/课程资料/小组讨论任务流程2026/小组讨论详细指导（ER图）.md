### 1.2 什么是实体、属性、联系？

根据教材（自学材料第2章 + 第7章PPT 7.3节）：

| 概念 | ER图符号 | 含义 | 本项目的例子 |
|------|---------|------|------------|
| **实体** | 矩形 | 现实世界中需要记录的"事物" | 学生、考试成绩 |
| **属性** | 椭圆 | 实体的特征/细节 | 姓名、分数、年龄 |
| **唯一属性** | 带下划线的椭圆 | 能唯一标识实体的属性 | student_id |
| **联系** | 菱形 | 实体之间的关系 | "拥有"（学生-成绩） |
| **基数约束** | 连线旁的符号 | 关系两端的数量限制 | 1:N、1:1、M:N |

**符号示例**（教材图7.9）：
```
        ┌─────┐
        │学生 │  ← 矩形 = 实体
        └──┬──┘
           │
     ┌─────┼─────┐
     │     │     │
  ┌──┴──┐ ┌──┴──┐ ┌──┴──┐
  │学号 │ │姓名 │ │性别 │  ← 椭圆 = 属性
  └─────┘ └─────┘ └─────┘
  (下划线表示唯一属性/主键)
```

### 1.4 基数约束的画法

根据自学材料第2.5节和第7章PPT，联系两端的基数约束用以下符号表示：

```
1:1 联系（学生—家庭背景）：
  学生 ──| |──<拥有>──| |── 家庭背景
         强制1个         强制1个

1:N 联系（学生—考试成绩）：
  学生 ──| |──<拥有>──→ 0── 考试成绩
         强制1个         0到多个

M:N 联系（教材例子：学生—课程选修）：
  学生 ──→ 0──<选修>──→ 0── 课程
         0到多个        0到多个
```

**符号含义**（自学材料图2-4）：
- `|`（竖线）= 1个
- `→`（箭头）= 多个
- `0`（圆圈）= 可选（可以有0个）
- `|`（竖线）= 强制（至少1个）

### 1.5 讨论流程建议

1. **腾讯会议开场**（5分钟）：明确讨论目标，分配记录人
2. **逐个实体讨论**（每个10分钟）：
   - 这个实体是否需要？有数据支撑吗？
   - 属性列全了吗？有主键吗？
   - 和其他实体的联系是什么类型？
3. **汇总确认**（15分钟）：确认所有实体、属性、联系无遗漏
4. **开始录屏**：为后续步骤2的表述录制作准备

### 2.3 每人局部E-R图详细设计

根据第7章PPT 7.3.5节和自学材料，每人按以下步骤画自己的局部E-R图。

#### 通用画法步骤

**步骤1：画出实体（矩形）**

**步骤3：画出联系（菱形）**

1. 选择"菱形"形状，写入联系名（如"拥有"）
2. 用直线连接到参与联系的两个实体
3. 在连线旁标注基数约束

**（2）命名冲突**

## 附：常见问题解答

### Q1：实体和属性怎么区分？

**原则**（第7章PPT 7.3.5节）：
- 如果一个"事物"有多个特征需要描述，且与其他"事物"有联系，就作为**实体**
- 如果一个"事物"只有单一值，不与其他"事物"有联系，就作为**属性**

**例子**：
- "考试成绩"是实体而不是学生的属性，因为成绩有科目、阶段、分数等多个特征，且一个学生有多条成绩
- "性别"是学生的属性而不是实体，因为它只有单一值（M/F），没有独立特征

### Q2：1:1联系和1:N联系怎么判断？

**判断方法**：
- 从A端看B端：A的一个实例最多对应B的几个实例？
- 从B端看A端：B的一个实例最多对应A的几个实例？

**本项目例子**：
- 学生—家庭背景：一个学生只有一条家庭信息，一条家庭信息只属于一个学生 → **1:1**
- 学生—考试成绩：一个学生有多条成绩，一条成绩只属于一个学生 → **1:N**

### Q3：画ER图时属性太多怎么办？

**建议**：
- 核心属性（主键+关键业务属性）画出来
- 次要属性可以省略或写在实体框旁边作为注释
- 最终E-R图可以只画主键和联系，详细属性在数据字典中说明
- 作业中建议画出所有属性，因为这是考核点

### Q4：局部E-R图和最终E-R图的关系？

- **局部E-R图**：每人画自己负责的1-2个实体及直接相关的联系
- **最终E-R图**：所有局部E-R图合并后的完整图，包含所有实体和联系
- 合并过程中要解决冲突、消除冗余
