# 1) 数据流

表示数据和数据流向，由一组固定成分的数据组成如“选课单”由“学号、姓名、课程编号、课程名”等成分组成 

数据流可从加工流向加工，也可在加工与数据存储或外部项之间流动；两个加工之间可有多股数据流 

数据流的命名 

用名词，不要使用意义空洞的名词 

尽量使用现实系统已有名字

# 2) 加工

表示对数据进行的操作，如“处理选课单”、“产生发票”等

尽量最好使用动宾词组，也可用主谓词组 

不要使用空洞的动词

# 1）DFD 的特性

与程序流程图不同，DFD 不表示程序的控制结构，只描述数据的流动 

DFD 分成多层（子图、父图概念）表示，从而逐步展开数据流和功能的细节。

# 2）画分层 DFD

(1) 先画出顶层 DFD。 

(2) 自顶向下画出各层 DFD 

分解原则： 

分解后的软件成分有相对独立功能 

一次分解不要加入细节过多 

由外向里画 DFD 

![](images/a349623942b2c91993841b85e5e120ca4dadba1df79c485ee3f78e1ca012042d.jpg)


# 2）画分层 DFD

(3) 先考虑稳定状态，忽略系统的工作条件，即怎么开始、怎么结束的。 

(4) 忽略琐碎的枝节，如出错处理等。 

(5) 随时准备重画

# 1）检查正确性

分析员可以从以下几个角度来检查DFD的正确性：

# 数据守恒

数据不守恒的情况有两种： 

一是某个加工输出的数据并无相应的数据来源，可能是某些数据流被遗漏了。 

![](images/4539f8bb831153e54f0d7a095a4614b7a7f1ddb781a59bb99221caffc7ed5dec.jpg)


运动员名单 = 队名 + 姓名 + 项目 

项目参加者 = 项目 + 姓名 + 运动员号 

“运动员号”并无数据来源 

# 数据守恒

二是，一个加工的输入并没有用到，这不一定是错误。可与用户进一步讨论，是否属于多余的数据流。 

![](images/72531c2c17ed5d04a75940dd7e541eb28e78ebab3a275da91d9598a1fd52394c.jpg)


订货单 = 单位名 + 货名 + 货号 + 数量 

价目 = 货名 + 单价 

发票 = 单位名 + 货名 + 数量 + 单价 + 总计经商量，“货号”确属多余，故删去。

# 分解的均匀

即图中各个部分不均匀。 

一张图中，如果某些加工已是基本加工（细节），而另一些加工还可进一步分解成三、四层，则应考虑重新分解。
