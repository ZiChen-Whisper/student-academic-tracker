# 📢 数据集调研任务通知

各位组员好，

我们的项目"**中小学学生学业发展动态跟踪与预警系统**"已进入前期数据集调研阶段。请大家各自寻找合适的数据集，并完成以下两项工作：

## 一、填写调研文档

1. 复制 `数据集调研模板.md` 到自己名字对应的目录下
2. 将文件重命名为 `数据集调研_你的姓名.md`（如 `数据集调研_Zichen_Zou.md`）
3. 按模板填写，重点关注"**基于该数据集可做的功能**"这一部分——根据你找到的数据集，思考我们能做哪些功能

## 二、下载数据集文件

将你找到的数据集下载后，放到 `前期准备/数据集/你的姓名/` 文件夹下。每人已建好对应文件夹：

- `前期准备/数据集/Zichen_Zou/`
- `前期准备/数据集/Zhiyan_Chen/`
- `前期准备/数据集/Zhuoxin_Liu/`
- `前期准备/数据集/Xinyi_Yu/`
- `前期准备/数据集/Zhen_Chen/`

## 几点提醒

- 数据集需要找**真实数据**（后续如果需要虚拟数据我们再加）
- 关注数据是否包含**成绩、考勤、作业、课堂表现**等学业相关字段
- 如果数据集文件较大（超过50MB），不要上传到仓库，可以上传到群文件里
- 截止时间：**2026.5.16 20:00**

完成后 push 到 docs 分支均可。

Commit 参考：（具体的Commit编写规范见Commit编写规范.md）

```bash
# 示例：提交调研文档
git add .
git commit -m "docs: 添加数据集调研文档 - Zichen Zou"

# 示例：提交数据集文件
git add student-performance.csv
git commit -m "data: 添加学生成绩数据集 - Zichen Zou"

# 示例：调研文档和数据集一起提交
git add .
git commit -m "docs: 添加数据集调研文档及数据集文件 - Zichen Zou"
```

> 注意：本项目统一使用**中文**编写 commit 信息，格式为 `类型: 描述 - 姓名`。
