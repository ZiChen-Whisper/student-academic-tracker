***

### 数据集概述

该数据集记录了学生的多维度学业信息，包含出勤率、两次内部测试成绩、作业得分、每日学习时长以及最终考试成绩。数据涵盖了学生学习过程的关键环节，从日常学习投入（学习时长、出勤）到阶段性测评（内部测试）再到最终考核（期末考试），形成了较完整的学业评价链条。适合用于学业成绩预测、学习行为与成绩关联分析以及学业风险预警等功能的开发。

### 数据规模

| 指标          | 数值                    |
| ----------- | --------------------- |
| **样本/记录数量** | 约1000+ 条记录            |
| **时间跨度**    | 单学期/单次记录              |
| **字段数量**    | 7 个维度                  |

### 主要字段

| 字段名                       | 含义         | 示例值      |
| ------------------------- | ---------- | -------- |
| Student_ID                | 学生唯一标识    | S1000    |
| Attendance (%)             | 出勤率（%）     | 84       |
| Internal Test 1 (out of 40) | 第一次内部测试成绩 | 30       |
| Internal Test 2 (out of 40) | 第二次内部测试成绩 | 36       |
| Assignment Score (out of 10) | 作业得分     | 7        |
| Daily Study Hours          | 每日学习时长（小时） | 3        |
| Final Exam Marks (out of 100) | 期末考试成绩   | 72       |

***

### 基于该数据集可做的功能

| 功能点              | 说明                                   | 依赖的关键字段                                             |
| ---------------- | ------------------------------------ | --------------------------------------------------- |
| 学业成绩预测            | 基于内部测试成绩、作业得分、出勤率等预测最终期末考试成绩     | Internal Test 1, Internal Test 2, Assignment Score, Attendance, Final Exam |
| 出勤率与成绩关联分析         | 分析出勤率与最终成绩的关系，识别出勤低但成绩尚可的"特殊"学生  | Attendance (%), Final Exam Marks                    |
| 作业完成质量分析           | 分析作业得分与最终成绩的关联，评估作业对学习的促进作用        | Assignment Score, Final Exam Marks                   |
| 学习效率评估             | 对比每日学习时长与成绩，识别学习效率高/低的学生群体          | Daily Study Hours, Final Exam Marks                  |
| 学业风险预警（**AI for DB**） | 综合出勤率、内部测试成绩下滑趋势、作业完成情况，预警潜在学业风险学生 | Attendance, IT1, IT2, Assignment, Final Exam       |
| 内部测试成绩趋势分析         | 对比两次内部测试成绩变化，识别成绩进步/退步学生             | Internal Test 1, Internal Test 2                     |
| NL2SQL自然语言查询（**AI for DB**） | 支持教师用自然语言查询如"出勤率低于85%且期末成绩低于70的学生" | 全部字段                                               |

***

### 数据质量

| 评估维度      | 情况                         |
| --------- | -------------------------- |
| **缺失值**   | 待检查，预计少量或无缺失值              |
| **数据真实性** | 基于学校教学管理数据整理，真实可信         |
| **其他问题**  | 数据为单时间点记录，无多阶段纵向数据；缺少课堂表现、家庭背景等字段 |

***

### 综合评价

**优势**：

1. 数据结构简洁清晰，字段定义明确，便于理解和使用
2. 包含出勤率、内部测试、作业、期末考试多个评价维度，覆盖学习全过程
3. 内部测试有两门（IT1、IT2）可做成绩趋势对比
4. 期末成绩为百分制，便于与其他数据集整合

**不足**：

1. 无多阶段纵向成绩数据，无法做长期学业趋势分析
2. 缺少家庭背景、学习习惯等社会人口统计学字段
3. 缺少学生姓名、班级等基本信息，不便于构建完整学生档案
4. 无时间维度，无法做历史成绩对比

## 二、数据集2：UCI学生学业表现数据集（Student Performance - Portuguese）

### 基本信息

| ------------ | -------------------------------------------------------------------- |

| **调研日期**     | 2026-05-15                                                             |
| **数据集名称**    | Student Performance (Portuguese Students)                              |
| **数据集来源/链接** | <https://archive.ics.uci.edu/ml/datasets/Student+Performance>          |

***

### 数据集概述

该数据集来自UCI机器学习库，记录了葡萄牙两所中学学生的学业表现及社会人口统计学信息。是本项目唯一具有多阶段成绩（G1、G2、G3）的数据集，可支持学业趋势分析这一核心功能。包含数学（student-mat）和葡萄牙语（student-por）两个学科数据，共33个维度的丰富信息，涵盖社会人口统计（性别、年龄、家庭背景）、学校因素（学校支持、课外辅导）、个人习惯（学习时间、休闲活动、健康状况）等多个方面。两个学科数据可做跨学科对比分析，识别偏科学生。

### 数据规模

| 指标          | 数值                         |
| ----------- | -------------------------- |
| **样本/记录数量** | 数学397条 + 葡萄牙语649条，共1046条记录 |
| **时间跨度**    | 一学年内的3个成绩周期（G1、G2、G3）      |
| **字段数量**    | 33 个维度                     |

### 基于该数据集可做的功能

| 功能点                         | 说明                                   | 依赖的关键字段                                              |
| --------------------------- | ------------------------------------ | --------------------------------------------------- |
| 学业趋势分析（**核心功能**）           | 利用G1→G2→G3三阶段成绩，分析学生成绩变化趋势（进步/退步/稳定） | G1, G2, G3                                          |

| 家庭背景对学业影响分析（**AI增强分析**）     | 分析父母教育水平、职业、家庭支持等对成绩的影响              | Medu, Fedu, Mjob, Fjob, famsup, G1-G3             |
| 学习时间与成绩关系分析               | 分析每周学习时间与成绩的关联                       | studytime, G1, G2, G3                              |
| 生活习惯影响分析（**AI增强分析**）        | 分析饮酒、外出频率、恋爱状态等生活习惯对成绩的影响            | Dalc, Walc, goout, romantic, G1-G3                |
| 家长参与度效果评估                 | 分析学校/家庭支持对学业的影响                    | schoolsup, famsup, G1-G3                           |
| 高等教育意愿与学业动力关联分析          | 分析有高等教育意愿学生的成绩特征                    | higher, G1-G3                                      |

| NL2SQL自然语言查询（**AI for DB**） | 支持教师用自然语言查询如"缺勤超过10次且G2成绩低于10的学生"   | 全部字段                                             |
| AI学业风险预警结果存储（**AI for DB**）  | 将AI预测的学业风险学生信息存入预警结果表               | risk_alert表结构                                     |
| AI个性化学习建议存储（**AI for DB**）   | 存储AI生成的学习建议及学生反馈                      | learning_suggestion表结构                            |

***

### 数据质量

| 评估维度      | 情况                                               |
| --------- | ------------------------------------------------ |
| **缺失值**   | 基本无缺失值，数据完整度高                                     |
| **数据真实性** | 真实数据，来自葡萄牙中学学生调查，UCI经典数据集，学术认可度高               |
| **其他问题**  | 样本量偏小（数学397+葡语649）；成绩为0-20分制；缺少学生ID无法跨学科关联同一学生 |

***

### 综合评价

**优势**：

1. **唯一具有多阶段成绩（G1、G2、G3）的数据集**，可做学业趋势分析和动态跟踪，是作业要求中学业跟踪预警系统的核心数据支撑
2. 字段维度最丰富（33个），包含社会人口统计、学校因素、学习习惯、健康状况等多方面信息
3. 两个学科数据可做跨学科对比分析和偏科识别
4. UCI经典数据集，有大量相关学术研究可参考，数据质量有保障
5. 包含缺勤次数、挂科次数等高风险指标，直接支持学业风险预警功能

**不足**：

1. 样本量较小，尤其是数学学科仅397条，可能影响模型泛化能力
2. 成绩为0-20分制，需要转换为百分制或做统一说明
3. 缺少学生ID，数学和葡语数据无法直接关联到同一学生
4. 无作业完成情况、课堂表现等过程性评价数据

## 三、数据集综合对比与功能映射

### 与作业要求功能对照

| 作业要求功能              | 数据集1（Final Marks）支持情况 | 数据集2（UCI Portuguese）支持情况 | 缺失及解决方案              |
| ------------------ | --------------- | -------------------- | ------------------- |
| 学业趋势分析             | ❌ 仅两次测试，无法做趋势   | ✅ G1/G2/G3三阶段成绩     | 数据集2已满足，无需补充        |
| 薄弱知识点识别            | ❌ 仅有总分，无科目细分     | ✅ 可对比数学和葡语成绩     | 数据集2已满足               |
| 学业风险预警（AI）         | ⚠️ 可做简单预警，缺少多因素  | ✅ 多因素支持预警模型构建    | 可用数据集2作为主要预警数据源     |
| NL2SQL自然语言查询（AI）   | ✅ 字段清晰可查询        | ✅ 字段丰富可多维度查询     | 两个数据集均支持              |
| 个性化学习建议（AI）        | ⚠️ 数据维度有限，建议较简单  | ✅ 因素丰富可生成针对性建议  | 建议以数据集2为主生成建议        |
| 学习风险预警结果存储（AI）     | ⚠️ 支持存储但因素少      | ✅ 预警信息更丰富         | 两个数据集均可，重点使用数据集2    |
| 成绩预测                 | ⚠️ 简单线性关系          | ✅ 可构建多因素预测模型     | 数据集2更优                 |
| 出勤率与成绩关联           | ✅ 出勤率为连续值        | ✅ 出勤用缺勤次数间接反映    | 数据集1更直观                |
| 作业完成情况              | ✅ 有作业得分            | ❌ 无                   | 可基于数据集2的famsup、failures等推断 |
| 课堂表现                | ❌ 无                 | ❌ 无                   | 可基于goout、freetime等习惯字段间接推断 |

### 综合优势

1. **数据集2是核心数据集**：唯一的多阶段成绩数据，是学业趋势分析功能的必要支撑；33个维度的丰富信息支持多因素预警模型构建
2. **数据集1是补充数据集**：出勤率、内部测试成绩等字段与数据集2互补；作业得分字段是数据集2缺失的
3. **两个数据集可整合**：期末成绩可统一为百分制；学生ID可通过学号规则生成实现跨数据集关联

## 四、综合评价

**优势**：

1. **数据集2的多阶段成绩（G1/G2/G3）是学业趋势分析的核心支撑**，是作业要求的必备数据
2. 两个数据集字段互补：数据集1提供作业得分和直观出勤率，数据集2提供多维度社会人口统计和生活习惯信息
3. 数据集2的UCI来源保证数据真实性和学术认可度
4. 成绩数据覆盖从日常测评（内部测试）到阶段性考核（G1/G2/G3）再到最终成绩的完整链条

**不足**：

1. 两个数据集均缺少作业完成情况和课堂表现的直接数据
2. 数据集2样本量偏小（尤其数学仅397条）
3. 两个数据集均无学生姓名等档案信息，不便于构建完整学生档案系统
4. 数据集2的成绩为0-20分制，需转换；数据集1为百分制，可作为转换参考
