# 数据集调研报告

## 1. 数据集基本信息

- 数据集名称：Student Performance Dataset
- 数据集来源：UCI Machine Learning Repository
- 数据集链接：https://archive.ics.uci.edu/dataset/320/student%2Bperformance
- 数据集类型：学生学业表现数据集
- 数据集适用方向：学生成绩分析、学业风险预警、学习行为分析、成绩预测

## 2. 数据集简介

Student Performance Dataset 是 UCI Machine Learning Repository 提供的学生学业表现数据集。该数据集主要记录了葡萄牙两所中学学生在数学和葡萄牙语两个学科中的学习表现。

数据内容包括学生的基本信息、家庭背景、学习时间、以往失败次数、是否参加课外辅导、出勤情况以及三个阶段的成绩。该数据集可以用于分析学生学业发展趋势，也可以用于构建学业风险预警模型。

## 3. 数据规模

该数据集包含两个主要数据文件：

1. student-mat.csv：数学课程学生表现数据
2. student-por.csv：葡萄牙语课程学生表现数据

其中 student-por.csv 包含 649 条学生记录，字段数量为 33 个。字段内容覆盖学生个人信息、家庭情况、学习行为、学校支持情况和成绩结果等方面。

## 5. 可用于本项目的功能设计

该数据集可以支持本项目中的多个核心功能：

### 5.1 学生成绩动态跟踪

可以利用 G1、G2、G3 三个阶段成绩展示学生成绩变化趋势。例如：

- G1 到 G2 上升，说明学生阶段性进步；
- G2 到 G3 下降，说明学生后期学习状态可能变差；
- G1、G2、G3 持续较低，说明学生存在较大学业风险。

### 5.2 学业风险预警

可以根据以下条件设置预警规则：

- 最终成绩 G3 低于 10 分，标记为高风险学生；
- 缺勤次数 absences 较高，标记为出勤风险；
- failures 大于 0，说明学生已有课程失败记录；
- studytime 较低且成绩下降，说明学习投入不足。

### 5.3 薄弱学生识别

通过成绩字段和学习行为字段，可以筛选出需要重点关注的学生。例如：

- 成绩连续下降的学生；
- 缺勤次数较多的学生；
- 学习时间短但失败次数多的学生；
- 缺少学校辅导和家庭支持的学生。

### 5.4 教师端数据分析

教师可以查看班级整体成绩分布、学生成绩变化趋势、风险学生名单和不同因素对成绩的影响。

### 5.5 家长端反馈建议

系统可以根据学生成绩、缺勤和学习时间生成简单建议。例如：

- 建议增加学习时间；
- 建议关注缺勤情况；
- 建议加强家庭学习支持；
- 建议参加学校额外辅导。

## 6. 数据质量分析

该数据集字段较完整，既包含成绩结果，也包含学习行为和家庭背景信息，适合做学业发展分析和预警模型设计。数据字段结构清晰，便于导入数据库和进行 SQL 查询分析。

但是，该数据集也存在一定局限：

1. 数据来源于葡萄牙中学，和国内中小学环境存在差异；
2. 数据字段中缺少具体班级、课程知识点、作业完成情况等信息；
3. G1、G2、G3 只有三个阶段成绩，动态跟踪粒度不够细；
4. 数据集规模中等，不适合直接代表所有学生群体。

## 7. 数据表设计思路

可以将该数据集设计为 student_performance 表，主要字段包括学生基本信息、家庭背景、学习情况、出勤情况和成绩结果。

后续系统开发时，可以在此基础上扩展为多张表，例如：

- student：学生基本信息表；
- score_record：成绩记录表；
- attendance_record：考勤记录表；
- warning_record：预警记录表；
- suggestion_record：个性化建议表。

## 8. 与项目主题的匹配度

该数据集与“中小学学生学业发展动态跟踪与预警系统”的匹配度较高。它可以支持成绩趋势分析、学业风险识别、缺勤影响分析和学习行为分析等功能。虽然数据不是国内中小学真实数据，但作为课程项目的数据基础比较合适。

## 9. 结论

综合来看，Student Performance Dataset 适合作为本项目的基础数据集。它字段丰富、结构清晰、与学生学业表现密切相关，可以支持系统中的学生成绩跟踪、风险预警和教师数据分析等功能。因此，建议本项目在前期原型设计中优先使用该数据集。
